In order to annotate a survey, you need to add an annotation set. An annotation set is exactly that - a set of annotations for your survey. It consists of its own image clustering information, and labels for those clusters etc. This means that you can have multiple annotation sets for a single survey, in turn allowing you to annotate a survey in many different ways (using different levels of AI species classification, for example), as well as import pre-existing annotations, and compare the results to determine both system performance, and what approach gives you the best speed-accuracy compromise for your particular application. It also allows you to do other things like have have new workers annotate old surveys to check their performance before letting them loose on your new data.

Additionally, each annotation set has its own set of labels, again allowing you to try different approaches to annotating the same data. Importantly, these labels can be created in a hierarchical structure with as many layers as you need. The intention of this approach is to allow you to have your species of interest at the top level of this hierarchy, with everything else dumped into group categories for annotation at a later stage when that infomation is required for, perhaps, another study. This in theory allows you to get all the infomation you need in as little time as possible.

So for example, if you are interested in large African predators (lion, leopard, cheetah, wild dog and hyena), then you want those labels at the top of your hierarchy. While you are at it, you may as well annotate the easily identifiable, "higher value" species like elephant, giraffe, and zebra, and them dump eveything else into categories like antelope and birds.

However, this approach has some other advantages in that is actually quicker to annotate in this layered way as you need to remember far fewer labels and associated hotkeys at a time, and you don't need to think about the more challenging categories. Additionally, this approach allows you to leave the challenging categories to experts to differentiate, rather than students or interns who may struggle.

When it comes to creating a new annotation set, there are two options: creating a new one, or importing pre-existing annotations from a csv file.

New Annotation Set

When you create a new annotation set, you can either create a new set of labels from scratch, or simply copy them from a previous annotation set (from any survey) by using the dropdown menus provided. There are also a number of templates provided for you, depending on your particular region/biome. You can also these templates or a copied label set as a good starting point for creating a new set of labels for a new biome.

When creating a new set of labels, keep the following in mind:

  • Begin with your top-level labels in your hierarchy by setting their parent to none, and then build downward.
  • You can also choose your own hotkeys that make sense for you, but the recommended approach is to use the label's first letter where there is no duplication (so that it can be remembered easily), and then to switch to numbers when there are two or more labels with the same first letter to avoid mistakes (eg. labelling a leopard as a lion).
  • You can easily add or remove labels by clicking the plus or cross buttons at any time.
  • You cannot have duplicate hotkeys within a particular category.
  • You cannot have duplicate label names anywhere in the hierachy.
  • Note that there will automatically be a 'Vehicles/Humans/Livestock' (v) category in the top level, along with a 'nothing' (n), a 'knocked down' (q), and an 'unknown' label (u). Also, there will be a 'skip' (0), and a 'wrong' (9) label included for all levels below that. As such, you will not be able to create labels with those names, or hotkeys.

CSV Import

In the case that you are importing annotations, all you need to do is to upload a csv file. The labels will be detected automatically and created in a flat hierarchical structure. The format for the csv columns is as follows: filename, label1, label2, .... Note the following:

  • Each row in the file should represent an image.
  • The filename is the full file path for the image within your folder structure, starting from your survey's top-level folder, as you uploaded it.
  • There may be any number of label columns, as long as they contain the word 'label'.
  • The labels themselves must be listed after the filename using commas, where no labels can be allocated using an empty space. Eg. lion,,,,
  • If there are no labels for an image, or an image is not contained within the csv file, it is assumed to be empty.
  • The first line of the file must consist of column names.
